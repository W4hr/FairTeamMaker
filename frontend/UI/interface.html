Project Goal

Welcome to FairTeamMaker, a platform designed to help you create balanced and fair teams based on player scores. This guide will walk you through the features of the website and explain how to use them effectively.

Tab Overview

The website is organized into six tabs, each serving a specific purpose in the team creation process. These tabs are arranged in the order you need to use them:

  • Introduction
  • Import Data
  • Edit Data
  • Analysis
  • Results
  • Logs (currently empty as a placeholder for future implementation)

Login and Account Management

To access all features, you need to log in:

  • Login: If you don't have a valid session token, you'll be redirected to the login page. You can log in with your username and password.
  • Sign Up: Create a new account with a username and password. Be careful with your password, as there's no password recovery option yet.
  • Session Duration: Your login is valid for 24 hours.

If you're not automatically redirected, please click here to go to the login page.

Import Data Tab

This tab allows you to access and manage your projects easily.

  1. Project Selection: On the left side, you can select a project saved on the server. New users have a default project they can load and edit. You can also create a new project using the "Add Project" button, where you can set a name, description, and color for easy distinction.
  2. File Upload: In the middle, you can upload a project file from your device. You can download your project as a JSON file at any time from the Analysis tab, edit it as needed, and then upload it here.
  3. Project Details: On the right, you can view and edit the project's details, such as the name and description, and delete players. Currently, this is the only way to delete players. You will also see a list of all players with their names and primary skill scores.

Edit Data Tab

In this tab, you can edit player data to refine how teams are formed.

  1. Attendance: The first column allows you to mark whether a player is present. Players marked as absent will not be considered in team formation.
  2. Player Names: The second column is for entering player names. Name changes here will automatically apply throughout the user interface.
  3. Primary Skill Score: The third column is for inputting the primary skill score, which is currently the only skill considered in the team balancing algorithm.
  4. Additional Categories: You can add more skill categories (e.g., strategy, endurance) by clicking the "+" button on the right side of the table. However, these additional categories are not yet implemented in the back-end and will not affect team balancing at this time.

Player-to-Player Relationships

You can open the Pair Performance table by clicking the corresponding button. This table allows you to define how well players perform together.

  • Structure: The table displays players' names both horizontally and vertically. Each cell represents the relationship between two players.
  • Mirrored Cells: Changes to one cell automatically update the corresponding mirrored cell.
  • Values:
    • Positive: Indicates improved performance when the two players are on the same team.
    • Negative: Indicates reduced performance when the two players are on the same team.
  • Custom Scale: The scale for these values is customizable.

Below the Pair Performance table, there is a "Discard Changes" button to revert to the last saved data. Next to it is the "Analyze" button, which takes you to the Analysis tab.

Analysis Tab

[Screenshot of the Analysis Tab with: 1 - Add Pitch Button, 2 - A Pitch, 3 - Settings]

This tab allows you to set up the environment and requirements for team formation.

1. Pitches

You can add pitches using the "Add Pitch" button. Each pitch contains two teams. For each pitch, you can:

  • Change the Pitch Name: Ensure each pitch has a unique name.
  • Change Team Names: Each team must have a unique name.
  • Define Team Size: Set the number of players per team. You can set it to "Automatic" to let the program determine the best team sizes.
  • Allocate Players: Assign specific players to teams. Players can be unallocated using the red "X" next to their names.

Note:

  • The number of allocated players cannot exceed the team size.
  • Each player can only be assigned to one team.
  • Teams cannot have the same name, and pitches cannot have the same name.

2. Settings

  • Interchangeable Teams: Toggle whether the teams should be balanced to be roughly equal in performance, allowing any team to play against any other.
  • Maximum Number of Sitting Out Players: Define how many players can sit out, which may result in fairer teams.
  • Maximum Difference Between Teams: Set the maximum allowable difference in the number of players between two teams.
  • Maximum Difference Between Pitches: Set the maximum allowable difference in the number of players between pitches.
  • Algorithm Selection: Choose between different algorithms for team formation:
    • Random: Tries various combinations and selects the best.
    • Brute Force: Tries a large number of combinations (recommended only for a small number of players).
  • Normalization Options:
    • Primary Score and Pair Performance: Toggle normalization on or off separately for each.
    • Normalization Methods:
      • Linear Normalization
      • Sigmoid Function
      • Logit Function
    • Minimum and Maximum Values: Define how values are normalized:
      • Symmetrical
      • Maximum Value
      • Minimum Value
      • Custom Value
    • Inversion: By selecting maximum for minimum and vice versa, you can invert the scale.
    • Weight: Set a weight for the Pair Performance relative to the Primary Score.

Normalization Explanation

Normalization adjusts different scales of data to a common scale, which helps in fair comparison and combination.

  • Linear Normalization:
    • Formula: Transforms values linearly to fit within a specified range.
    • Use Case: Simple scaling without altering the distribution of data.
  • Sigmoid Function:
    • Formula: \( f(x) = \frac{1}{1 + e^{-x}} \)
    • Characteristics: Compresses extreme values towards the center of the scale.
    • Use Case: Reduces the impact of outliers, emphasizing moderate values.
  • Logit Function:
    • Formula: \( f(x) = \ln\left(\frac{x}{1 - x}\right) \)
    • Characteristics: Expands values at the extremes while compressing those near the center.
    • Use Case: Highlights significant differences in extreme values.

3. Actions

  • Download Project: Save your project as a JSON file.
  • Save Project: Save your project to the server.
  • Analyze: Send the project to the server for calculation. The analysis may take a few seconds.

Note:

  • Validation: Before any action, the data is validated. If an action doesn't execute, it's likely due to a validation error, which will be displayed.
  • Error Reporting: Errors can be reported at GitHub Issues.

Results Tab

[Screenshot of the Results Tab with: 1 - Options List, 2 - Pitches and Teams]

After analysis, you will be redirected to the Results tab.

1. Options List

On the right, you'll see a list of possible team combinations. Each option previews:

  • Total performance difference between teams.
  • Number of players sitting out.

2. Detailed Results

Selecting an option displays:

  • Pitches and Teams: Shows the pitches and the teams on each pitch.
  • Allocated Players: Lists players assigned to each team, along with their normalized scores.
  • Team Scores: Overall score for each team based on the allocated players.
  • Performance Difference: The difference in performance between the two teams on each pitch.
  • Team Sizes: Number of players in each team.
